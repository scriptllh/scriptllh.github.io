redis事务

2018-06-05
redis
约 439 字 预计阅读 1 分钟 次阅读

文章目录

  1. 事务

    • redis 的命令是一组命令的集合

    • redis 保证了一个事务中的所有命令要么执行，要么不执行，如果发送Exec命令前客户端断线了，则redis会清空事务队列，事务中的所有命令不会执行，而点客户端发送了exec命令，所有命令都会被执行，即使客户端断线

    • 除此之外，Redis的事务还能保证一个事务内的命令依次执行而不被其他命令插入。试想客户端A需要执行几条命令，同时客户端B发送了一条命令，如果不使用事务，则客户端B的命令可能会插入到客户端A的几条命令中执行。如果不希望发生这种情况，也可以使用事务。

    • 和传统事务不同，一组命令中途执行发生了错误不能回滚（一般采用日志记录然后业务补偿，一般情况下redis不能有这种强一致性的要求）

  2. watch

    • WATCH命令可以监控一个或多个键，一旦其中有一个键被修改（或删除），之后的事务就不会执行。监控一直持续到EXEC命令
    • 由于WATCH命令的作用只是当被监控的键值被修改后阻止之后一个事务的执行

  3. 解决监听机制的一种思路

    • 一端写数据data加变动的key
    • 客户端一直轮询变动的key
